```

       │  RESTful API (axios)
       ▼
post-robot-system/ (Spring Boot 3.x + MyBatis-Plus)
┌────────────────────────────────────────────┐
│  controller/  接口层 (11 Controllers)       │
├────────────────────────────────────────────┤
│  service/     业务服务层                     │
├────────────────────────────────────────────┤
│  adapter/postal/  邮政对接适配层             │
│  ├─ spi/PostalApi           接口契约         │
│  ├─ spi/PostalApiMock       Mock 实现       │

├────────────────────────────────────────────┤
│  mapper/  MyBatis-Plus 数据访问 (11 Mapper) │
├────────────────────────────────────────────┤
│  model/  实体 + 枚举 + 请求/响应 DTO        │
```

## 数据模型

11 张表，均含主键 (Snowflake)、create_time、update_time、is_deleted；金额 DECIMAL(10,2)，状态字段枚举注释。

| 表 | 核心字段 |
|---|---------|
| product | name, price, tags, robot_graspable, display_point |
| inventory | real_stock, locked_stock, low_stock_threshold, sample_status, mismatch_flag |
| orders | order_no, total_amount, status, qr_code_url, payment_flow_no, version(乐观锁) |
| order_item | order_id, product_name/snapshot, price/snapshot, quantity, subtotal |
| payment | payment_flow_no, status, qr_code_url, pay_query_no, platform_flow_no |
| order_status_log | from_status, to_status, operator, operation_type |
| payment_flow_record | flow_type, resp_data_digest, source_ip |
| task | task_no, task_type, status, priority(1-10), dependency_task_no, timeout_seconds, retry_count |
| alert | alert_type, alert_level, source, source_id, status, handler |
| user | username, password, role(ADMIN/OPERATOR/MAINTAINER) |
| audit_log | operator, operation_type, target_type, result, trace_id |

## 对接适配层设计

### 报文格式

```
YYRoot<T> {
  sessionHeader: {
    serviceCode: "F1"～"F5",          // 服务代码
    version: "YY-1.0",                // 版本
    actionCode: "0",                  // 操作代码
    transactionId: "POSTR+yyyyMMddHHmmssSSS+0000000000",  // 32位事务ID
    srcSysId: "POSTR", dstSysId: "XYDYYQDXT",
    digitalSign: "BASE64(MD5(拼接串+Secret))",   // 签名
    reqTime: "yyyyMMddHHmmss"
  },
  sessionBody: T   // 业务数据（泛型）
}
```

### 签名算法

拼接顺序：`ServiceCode + Version + ActionCode + TransactionID + SrcSysID + DstSysID + ReqTime + SessionBody(JSON) + Secret` → MD5 → BASE64。

### 5 个接口

| 接口 | 服务代码 | Mock 状态机 |
|------|---------|-------------|
| 资费查询 | F1 | 固定 12.00 元 |
| 号码生成 | F2 | 自增唯一号码 |
| 收寄提交 | F3 | 返回交易流水号 |
| 二维码 | F4 | 初始化支付状态 |
| 支付查询 | F5 | 首次→支付中(00)，延迟后→支付成功(01) |

### Mock / 生产切换

`postal.mock.enabled=true/false`，业务代码通过 `PostalApi` 接口调用，不感知差异。

## 关键机制

| 机制 | 技术方案 |
|------|---------|
| 库存并发控制 | Redisson 分布式锁 + Lua 脚本（原子 DECRBY/INCRBY） |
| 订单并发控制 | MyBatis-Plus @Version 乐观锁 |
| 订单状态机 | OrderStateMachine（7 状态，完整迁跃矩阵） |
| 支付幂等 | 双重检查：payment.status + order.status |
| Redis-DB 一致性 | TransactionSynchronization 回滚补偿 |
| 审计日志 | @AuditLog AOP（自动捕获操作人/类型/对象/IP/结果） |
| 鉴权 | JWT access+refresh 双令牌 + Redis 黑名单 |
| 幂等 | 0→1→2→3→4→5→6→7→8 |

## 前后端覆盖

后端 61 个 API 端点，前端 11 个页面，60 个 API 函数全覆盖（健康检查除外）。

前端技术栈：Vue 3 + Element Plus + Pinia + Vue Router 4 + Axios。
后端技术栈：Spring Boot 3.5 + MyBatis-Plus 3.5 + Spring Security 6 + Redis/Redisson + Flyway + jjwt 0.12。
